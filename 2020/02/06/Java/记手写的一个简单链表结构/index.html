Java数据机构-记·手写的一个简单链表结构

链表的特点：

链表是以分散的形式存储在内存中的，在查询元素时只能通过上个元素中存储的下个元素的信息进行查询。也就是说，链表不支持随机访问，要想查询某个元素，只能从头节点开始遍历。时间复杂度相较于数组的随机访问会增大；

详细的可参照：Java数据机构-链表特性

#####单向链表：
1.首先得定义Node节点及相关结构

1)节点有两个属性（因为实现的是一个单向链表结构所以只定两个属性）：

a>当前节点的value
b>下个节点

2.链表的实现类：
为了方便能够访问私有属性，所以Node是LinkedList(实现类)的内部类；
链表结构很像“火车“，每节车厢对应的是每个Node节点，而连接每节车厢的那个链条就是我们需要处理的引用传递，而如果我们想要这个火车跑起来那必须得有个火车头，那么我们就需在定义个头节点和链表的size（火车一共有多少节车厢）；

实现思路：
1.首先查找到指定索引的Node节点
2.然后将 新节点的下个节点指向前节点的下个节点 ，再将 前节点的下一个节点指向新节点

e> 根据指定的索引删除：

实现思路：
1.根据索引查找对用节点
2.将 该节点 的上个节点指向 该节点 的下个节点
3.将 该节点 的 下个节点 指为 null

f> 根据指定值删除

实现思路：
1.根据 value 值 查找 对应 索引
2.调用 removeIndex 方法 删除节点

g> 链表反转（这个就有点意思了）

实现思路：
1.首先的理清思路，既然是链表全部都反转那么就是只需要将 当前节点的后节点指向前节点就ok了；
2.所以我们就需要 获取的 当前节点的前节点 当前节点 当前节点的后节点， 再将 当前 引用关系 反转；
3.最后再将 未节点 变成 头节点 ；

h>将指定的元素 和 指定的索引元素 互换（也蛮有意思）

实现思路：
1.首先根据元素找到 该元素 及 该元素的前节点
2.再根据索引找到 该元素 及 该元素的前节点
3.如果是 头节点 就将 head 设为 将要交换的节点

i>根据index获取Node元素

3.总结：

1.缺点：单链表相关实现上比较单一且清晰，但是在进行查询元素的时候因为没有前驱节点的信息无法做相关的优化处理 及 在 交换 元素的时候，也会多一些实现的逻辑，代码就显得不够优美，在性能上较双向链表也会显得不足；
2.优点：如上面所说如果每个节点都存有上个节点的信息，那么必然在内存空间利用上就劣与单链表；
