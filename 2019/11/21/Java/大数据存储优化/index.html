数据量 10W 100W 1000W

1000W数据时会报错：

这是在快OOM之前JVM的保护措施，代表垃圾回收耗费了98%的时间，但是回收的内存还不到2%，那么JVM会认为即将发生OOM，让程序提前结束。

可以看出，JdbcTemplate的存储性能确实远远好于Jpa，但是也有瓶颈；在上百万级别的数据可以考虑两种方案进行优化：
redis和多线程批量保存。这两种方案在后面我做好数据统计会再发出来，今天暂不赘述；
